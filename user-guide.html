FilterableList.js Beginner Guide

This guide helps you set up and use the plugin with clear copy/paste examples.

1) What this plugin does

  • Filter items by values such as type, level, or category.
  • Sort items by a selected attribute.
  • Reverse visible item order.
  • Reset everything to the original state.
  • Show a no-matches message when nothing matches.

2) Add the plugin files

<link rel="stylesheet" href="filterable-list.css" />
<script src="filterable-list.js"></script>

Place the script near the end of the body, or keep it in the head and rely on DOMContentLoaded.

3) Use the required HTML structure

Required pieces:

  • A wrapper with class filterable-container
  • Controls with elements using data-action
  • A list with class filterable-list
  • Items with class item and data-properties
<div class="filterable-container">
  <div class="filter-controls">
    <button data-action="reset" class="active">Reset</button>
    <button data-action="filter" data-filter-attribute="type" data-filter-value="painting">Painting</button>
    <button data-action="sort" data-filter-attribute="price">Sort by Price</button>
    <button data-action="reverse">Reverse</button>
  </div>

  <ul class="filterable-list">
    <li class="item" data-properties="name-Sunrise type-painting level-beginner price-25">Sunrise</li>
    <li class="item" data-properties="name-Cityscape type-photo level-intermediate price-40">Cityscape</li>
    <li class="no-matches-message">No items match your current filters.</li>
  </ul>
</div>

4) How data-properties works

Format each item as key-value pairs separated by spaces:

name-Sunrise type-painting level-beginner price-25
  • Matching is case-insensitive.
  • Numbers are parsed as numeric values when possible.
  • Use consistent key names across all items and controls.

5) Control actions

Use these values for data-action:

  • filter: needs data-filter-attribute and data-filter-value
  • sort: usually needs data-filter-attribute
  • reverse: no extra attributes needed
  • reset: no extra attributes needed

6) Supported input types

  • Links and buttons use click events.
  • Checkbox and radio use change events.
  • Select uses change events.
  • Text/search input uses input events.

7) Optional global configuration

<script>
  configureFilterableList({
    hiddenClass: 'hidden',
    activeFilterClass: 'active',
    enableTransitions: true,
    transitionDuration: 400
  });
</script>

8) Common beginner mistakes

  • Forgetting class filterable-container.
  • Missing class item on list entries.
  • Typing a control attribute key that does not exist in items.
  • Using values that do not exactly match item values.

9) Quick checklist

  • Include both plugin files.
  • Create a filterable container with controls and list.
  • Add data-properties on each item.
  • Test reset, filter, sort, and reverse.
